# 📘 Assignment: Building REST APIs with FastAPI

## 🎯 Objective

Learn how to design and implement RESTful APIs using the FastAPI framework in Python. Students will gain hands-on experience with endpoints, CRUD operations, and basic API design principles.

## 📝 Tasks

### 🛠️ Task 1: Set Up FastAPI Project

#### Description
Initialize a new FastAPI project and set up the basic application structure.

#### Requirements
Completed program should:
- Install FastAPI and Uvicorn
- Create a main application file (main.py)
- Set up a basic FastAPI app and run a test server

### 🛠️ Task 2: Implement CRUD Endpoints

#### Description
Create RESTful endpoints for a resource (e.g., books, users, or items) that support Create, Read, Update, and Delete operations.

#### Requirements
Completed program should:
- Define a Pydantic model for the resource
- Implement endpoints for POST (create), GET (read), PUT (update), and DELETE (delete)
- Store data in an in-memory list or dictionary
- Return appropriate HTTP status codes and responses

### 🛠️ Task 3: (Optional) Add Extra Features

#### Description
Enhance your API with additional features such as input validation, error handling, or authentication.

#### Requirements
Completed program could:
- Validate input data and handle errors gracefully
- Add simple authentication (e.g., API key or token)
- Document endpoints using FastAPI's automatic docs

---

**Skills practiced:** FastAPI, REST API design, CRUD operations, Python, Pydantic models, HTTP methods

**Tip:** Use FastAPI's interactive docs at /docs to test your endpoints!
